## <a name="spatialization"></a><span data-ttu-id="2fd8d-179">Spatialization</span><span class="sxs-lookup"><span data-stu-id="2fd8d-179">Spatialization</span></span>
<span data-ttu-id="2fd8d-180">Spatialization 使用立体声耳机或扬声器将声音置于混合世界。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-180">Spatialization uses stereo headphones or speakers to place sounds into the mixed world.</span></span>

<span data-ttu-id="2fd8d-184">Spatializing**用户界面**元素可帮助整理用户的 sonic "space"，方法是将已锁定的立体声声音的数目限制为其打印头。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-184">Spatializing **user interface** elements helps declutter the user's sonic "space" by limiting the number of stereo sounds locked to their heads.</span></span> <span data-ttu-id="2fd8d-185">特别是在直接操作交互中，在 spatialized 音频反馈时，接触、抓取和释放感觉更自然。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-185">Especially in direct manipulation interactions, touching, grabbing, and releasing feels more natural when audio feedback is spatialized.</span></span> <span data-ttu-id="2fd8d-186">但请参阅下面有关这些元素的距离衰减情况。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-186">However, see below regarding distance attenuation for these elements.</span></span>

<span data-ttu-id="2fd8d-187">Spatializing**视觉对象指示器**和所介绍的 **AI 声音**直观地通知用户在视图外的情况。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-187">Spatializing **visual indicators** and **_embodied_ AI voices** intuitively informs users when they are outside the field of view.</span></span>
    
<span data-ttu-id="2fd8d-188">与此相反，应避免 spatialization 用于 **_faceless_ AI 声音**，并避免使用未定义好的空间位置的其他元素。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-188">In contrast, avoid spatialization for **_faceless_ AI voices**, and other elements without a well-defined spatial location.</span></span> <span data-ttu-id="2fd8d-189">如果没有相关的视觉元素，Spatialization 可能会分散用户认为有一个视觉元素无法找到。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-189">Spatialization without a related visual element can distract users into thinking there is a visual element that they can't find.</span></span>

### <a name="when-and-how-should-i-apply-distance-based-attenuation"></a><span data-ttu-id="2fd8d-194">何时以及如何应用基于距离的衰减？</span><span class="sxs-lookup"><span data-stu-id="2fd8d-194">When and how should I apply distance-based attenuation?</span></span>

<span data-ttu-id="2fd8d-198">与**视觉对象指示器**、**动画影像**和其他信息性声音的距离通常与用户相关。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-198">The distances to **visual indicators**, **animated holograms**, and other informative sounds are usually relevant to the user.</span></span> <span data-ttu-id="2fd8d-199">使用基于距离的衰减直观地提供此提示。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-199">Use distance-based attenuation to intuitively provide this cue.</span></span>
* <span data-ttu-id="2fd8d-200">调整每个源的衰减曲线，使其适应混合世界空间的大小。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-200">Adjust the attenuation curve for each source to fit the size of your mixed world spaces.</span></span> <span data-ttu-id="2fd8d-201">音频引擎的默认曲线通常适用于非常大（最多为半 kilometer）空间。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-201">Your audio engine's default curve is often meant for very large (up to half-kilometer) spaces.</span></span>

<span data-ttu-id="2fd8d-202">强调按钮和其他交互的**渐进式阶段**的声音不应应用衰减。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-202">Sounds that reinforce the **progressive stages of buttons** and other interactions shouldn't have attenuation applied.</span></span> <span data-ttu-id="2fd8d-203">这些声音的加强效果通常比与按钮通信的重要性更重要。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-203">The reinforcing effects of these sounds are generally more important than communicating the distance to the button.</span></span> <span data-ttu-id="2fd8d-204">变体可能会让人分散注意力，尤其是在键盘上，很多按钮单击将会连续听到。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-204">Variations can be distracting, especially with keyboards, where many button clicks will be heard in succession.</span></span>

### <a name="which-spatialization-technology-should-i-use"></a><span data-ttu-id="2fd8d-205">我应该使用哪种 spatialization 技术？</span><span class="sxs-lookup"><span data-stu-id="2fd8d-205">Which spatialization technology should I use?</span></span>
<span data-ttu-id="2fd8d-206">使用耳机或 HoloLens 扬声器时，请使用基于 HRTF （head 相关传输函数）的 spatialization 技术。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-206">When using headphones or the HoloLens speakers, use HRTF (head-related transfer function)-based spatialization technologies.</span></span> <span data-ttu-id="2fd8d-207">它们对现实世界内的声音传播建模。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-207">They model the sound propagation around the head in the physical world.</span></span> <span data-ttu-id="2fd8d-208">即使在一侧的一侧有一个声音源，也会出现一些衰减和延迟的情况。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-208">Even when a sound source is far on one side of the head, sound propagates to the distant ear with some attenuation and delay.</span></span> <span data-ttu-id="2fd8d-209">相反，发言人摇摄仅依赖于衰减，在右侧的声音（反之亦然）时，它会在左侧的耳中应用总衰减。</span><span class="sxs-lookup"><span data-stu-id="2fd8d-209">Speaker panning, in contrast, relies only on attenuation, and applies total attenuation in the left ear when sounds are on the right side (and vice-versa).</span></span> <span data-ttu-id="2fd8d-210">对于正常听觉的侦听器，这种情况可能会令人感到不安，对于在一个 ear 中具有听力障碍的侦听器，此情况可能不</span><span class="sxs-lookup"><span data-stu-id="2fd8d-210">This can be uncomfortable for normal-hearing listeners, and inaccessible for listeners with hearing impairment in one ear.</span></span>
