# <a name="case-study-expanding-the-design-process-for-mixed-reality"></a>案例研究：展开混合现实的设计过程

根据 Microsoft 在 2016 年推出到预先开发人员的受众 HoloLens，团队已使用工作室来生成设备的启动体验的 Microsoft 内部和外部合作。 这些团队，了解通过这样做，在混合的现实设计的新字段中查找机遇和挑战。

为了帮助新团队和合作伙伴更有效地进行创新，我们启用其方法并能够深入了解的设计和开发的课程我们为开发人员在我们混合现实大学课程 (包括为期一周的设计研讨会向我们提供我们企业合作伙伴）。

下面是这些知识付，我们使用来帮助我们用于混合的现实开发准备其团队的企业合作伙伴较大过程的一部分的快照。 尽管其中许多方法最初针对 HoloLens 开发，ideating 和通信以三维形式是至关重要的整个范围的混合的现实体验。

## <a name="thinking-spatially-during-the-design-process"></a>在设计过程中在空间上考虑

任何设计过程或设计框架用于循环访问考虑：广泛地解决问题，若要有效地与他人交流想法，以评估这些想法并访问解决方案。 今天，我们已获得广泛认可的设计和开发方法，用于在台式计算机、 手机和平板电脑上构建的体验。 团队还可以清晰的循环访问想法并将其转换为用户的产品所需的期望。

通常团队多种开发、 设计、 调查和管理，但所有角色可以都参与设计过程。 若要提供移动应用程序了解的门槛可以绘制一个矩形的设备的屏幕一样简单。 草绘框和线的 UI 元素，使用箭头以指示动作或交互时可以便足以建立技术要求或定义潜在用户行为。

混合现实与传统的 2D 设计过程开始崩溃： 草绘以三维形式是大多数人难以并使用 2D 工具，例如笔和纸或白板，可以经常会限制为那些尺寸的想法。 同时针对游戏或工程，生成的 3D 工具需要能够熟练地快速充实想法的高度。 缺少的轻量工具之外，还有通过固有的技术不确定性有新的设备，仍建立基础交互方法其中。 这些挑战可能会限制你的团队只允许具有三维开发背景的设计贡献，这大大减少了迭代团队的能力。

![在中混合现实 Academy 研讨会期间处理的团队](images/academyTeam1000.png)<br>
*从我们的研讨会中的混合现实合作伙伴程序团队*

当我们与外部合作伙伴合作时，我们听到等待完成原型帮助开发人员的团队的情景可以继续其设计过程之前。 这通常意味着团队的其余部分以有意义在方面取得进展的产品，重载解决同时技术实现，以及用户的主要组件开发人员体验 （因为它们都要放入代码的粗略） 被阻止。

## <a name="techniques-for-expanding-the-design-process"></a>扩展设计过程的方法

我们在 Microsoft 的团队具有一系列技术来更有效地包括他们的团队并快速循环访问复杂的设计问题。 时未一个正式的流程，这些技术旨在补充而不是替换您的工作流。 这些方法允许那些没有专门的 3D 技能在深入探讨原型制作阶段之前提供的想法。 不同的角色和领域 （而不仅限于三维设计人员和开发人员） 可以是设计过程中，同机会和确定可能更高版本在开发中可能出现的挑战的一部分。

## <a name="generating-ideas-with-bodystorming"></a>生成与 bodystorming 意见

获取你的团队考虑一下在现实生活中发生的事件，超出了传统的 2D 设备世界是关键开发创新的混合的现实体验。 在 Microsoft，我们发现执行此操作的最好办法是鼓励与现实世界空间中的物理属性的交互。 我们使用简单、 廉价构建材料生成物理属性来表示数字的对象、 用户界面和建议的体验中的动画。 此技术称为 bodystorming，已被一个主要方面数十年时间内工业设计产品构思。

![团队使用的材料中混合现实学院](images/academyWork1000.png)
*bodystorming 中使用的简单、 廉价的艺术作品提供*

简单的物理属性级别游戏区参与者，允许使用不同的技能和背景个人提出自己的想法并发现不会拘泥的 2D 模式混合的现实体验固有的机遇考虑。 虽然技术原型设计和实现高保真情节提要需要熟练的 3D 开发人员或艺术家，几个聚苯乙烯球和 cardboard 可以是不足以展示如何接口可能会展开的物理空间。 这些技术适用于使用 HoloLens 混合的现实开发和沉浸式耳机。 例如，一组塑料连接器可能大致说明了全息 HoloLens 体验或属性来执行种交互元素或运动设计虚拟世界中的出现的大小。

Bodystorming 是一种技术来快速生成的想法和评估是原型太含糊的想法。 在 Microsoft，bodystorming 通常用于快速检查想法，但也可以是托管的更深入的会话，如果涉及外部利益干系人还不熟悉混合的现实开发或需要提取并将其涉及范围非常广泛的方案很有帮助。 请记住 bodystorming 旨在快速高效地通过 ideate 鼓励参与者想在空间上。 复杂的图片或精确的测量值在此阶段不重要。 物理属性只需要满足的最低要求，将探索或交流想法。 不应通过 bodystorming 表达的概念已经过全面审查，但该过程可以帮助缩小可能性更高版本在设备在原型设计阶段测试。 在这种情况下，bodystorming 不应替换技术原型制作，但而不是偏移量解决这两种技术的负担，并在原型制作阶段设计难题。

## <a name="acting-and-expert-feedback"></a>活动和专家的反馈

按照 bodystorming ideating 与现实世界中的物理对象的过程下, 一步是遍历这些对象的体验。 我们调用此阶段中的过程操作，并且它通常涉及到暂存体验或特定的交互通过移动用户将如何。

![团队在中混合现实 Academy 研讨会期间操作方案](images/academyCollab1000.png)<br>
*团队在研讨会期间操作方案*

与物理属性使其执行操作使参与者不仅体验通过用户的角度考虑，但允许外部观察者以查看事件的播放方式。这会带来的理想时间，以包括更多的人的团队成员或利益干系人能够提供特定的专业反馈。 例如，如果在探索为医院设计的混合的现实体验，操作出您的想象力医疗专业人员可以提供非常宝贵的反馈。 另一个示例是当你可能想要了解，如空间音频或预算资产质量与性能的一个特殊的挑战。 在这些情况下，使其执行操作会让专家们如何体验可能会将展开，而无需设备基于原型的快速、 粗略的概念。

未正式这种操作，而无需引入专业人才，但在我们想要与他人共享想法不的情况下存在视频，我们将记录场景的交互或包括情节提要艺术家，若要观察并草图密钥时间。 操作也可以是非常轻量的活动，通常发生就地 bodystorming 阶段。 确定要使用取决于受众 （和引出的反馈的必要的类型所需的保真度），但最终取决于任何最有效地将捕获团队的想法。

## <a name="capturing-ideas-with-storyboards"></a>捕获使用演示图板的想法

传达的想法和建议的体验，您的概念的最佳方法取决于您的目标的受众，以及下一次迭代需要的反馈的类型。 向团队成员显示新的想法时, bodystorming 的低保真度重新 enactments 可以是足以将加快人。 当引入新的利益干系人或潜在用户体验概念，最好的方法通常情节提要。 情节提要是通用的娱乐行业，在电影和视频游戏开发，通常在后台中找到一种技术，可帮助传达这两个的总体过程的一种体验 （在低 fidelities) 以及 （在高美观的外观和感觉fidelities)。 就像使用原型制作一样了解你的情节提要的保真度需求是反馈的收集正确类型和避免使系统效率降低讨论的关键所在。

![低保真情节提要的示例](images/storyboardLow1000.png)<br>
*低保真情节提要的示例*

低保真 storyboard 是快速讨论的右保真度，尤其是当传达高级概念。 这些可以是简单的线条图绘图和基元形状来表示在场景或交互组件 （物理和虚拟） 邻近虚拟元素。 虽然这些都是有用给定相对轻松地和低技能屏障来执行，请从 bodystorming 记住在课：并非所有人都可以看到 2D 形式表示的一种体验，并了解三维的含义。

![高保真情节提要的示例](images/storyboardHigh1000.png)<br>
*高保真情节提要的示例*

高保真 storyboard 或与您的建议美观方向相结合 bodystorming 会话的见解时引入新的利益干系人是体验的一个强大的工具。 情节提要可以构建心情看板以并列的虚拟体验的最终外观，以及捕获可能会对最终产品的关键时间点。 请记住的高保真情节提要通常需要艺术家，尤其是嵌入在团队中可以捕获很难描述的想法。 在 Microsoft，我们已添加情节提要艺术家，通常与背景的游戏开发，到我们的混合的现实团队出席会议做好快速捕获更高版本将扩展到更高的保真度模型的草图的人员。 与技术艺术家，帮助传达对最终体验中使用的资产的艺术方向，这些人员紧密合作。

## <a name="expanding-your-teams-skills"></a>展开您团队的技能

除了所述，更改你的过程以便更好地适应混合的现实的技术体验取决于更多的紧密连接技术和设计角色。 在 Microsoft 与合作伙伴合作，我们见证到多个团队成功过渡到三维开发，并查找最大优势来自团队成员跳出绰绰有余，开发技能，为他们提供更多的参与在整个开发过程 （不只限制自己的技能与设计或开发）。

反馈和迭代是任何成功设计、 混合的现实体验到更多技术概念，尤其是考虑混合的现实的技术和工具的相对 nascence，则通常会这种反馈关键字。 构建您的设计团队的技能在技术领域，包括精通 Unity 或 Unreal 等工具可帮助更好地了解开发人员的方法和原型制作过程中提供更有效的反馈的个人。 同样，开发人员了解混合的现实体验 （和潜在的设计缺陷） 的基本用户体验概念可以帮助他们规划的早期阶段提供实现见解。

我们通常从合作伙伴在有关如何混合现实中最适合使用增大其团队获得问题并与 3D 工具技能。 考虑这一点很难回答的问题的状态的行业，如团队面临着构建明天的体验与昨天的工具。 许多团队收集的个人才能背景的游戏和娱乐，其中 3D 开发具有数十年来已处装订，或鼓励他们现有的团队选取 Unity 和 3D 建模软件之类的工具。 虽然这些都是立即满足基线的混合的现实体验这两种基本的需求，很有可能将大量专用的工具和技能，帮助构建明天的创新应用程序。

在不久的将来，更多的工作重点放在全息版体验中的数据和云服务，混合的现实设计的字段将发生显著更改。 这一体验的下一代将充分利用高级的自然语言处理和实时计算机视觉、 以及本身更富有社交方案中沉浸式体验。 例如，虚拟现实中的体验，设计人员可以构建创新型环境和空格，需要设置更像是体系结构或心理学而不是传统的界面设计的一项技能。 或者，考虑的 HoloLens，其中用例通常涉及高度专业化的字段，例如事故，工程、 或生产，其中特定环境和实时数据体验的必备要素与现实世界中的体验。 在角色中工作，利用设计和专业的知识，并将乐于学习新工具是为在混合现实中工作的团队非常有用的技能。

## <a name="helping-your-team-explore-mixed-reality-quickly-and-effectively"></a>帮助您快速有效地浏览混合的现实的团队

在 HoloLens 的早期阶段，这些技术的载体不需要为设备的早期原型硬件被证明是一个小于理想格式用于快速遍历设计理念。 交互方法，包括在操作系统中的基本 UI 这探索出了与 bodystorming，而使其执行操作帮助我们了解在工作区中的方案的基本用户行为的可用性团队。 最后，团队就能够建立的基线强的用户体验的 HoloLens，跨团队角色通信的概念，允许与硬件开发并行快速移动的操作系统开发的核心。
