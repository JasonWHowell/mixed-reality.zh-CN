# <a name="shared-experiences-in-mixed-reality"></a>混合现实中的共享体验

全息无需保持私有的只是一个用户。 全息版的应用程序可能会共享[空间的定位点](coordinate-systems.md)从一个 HoloLens、 iOS 或 Android 设备，使用户能够跨多个设备呈现一张全息图现实生活中相同的位置。

## <a name="six-questions-to-define-shared-scenarios"></a>六个问题以定义共享的方案

在开始设计的共享体验之前，务必定义目标方案。 这些方案将帮助阐明您正在设计并建立的常见词汇来帮助进行比较和对比你的体验中所需的功能。 了解核心问题和解决方案的不同的途径，是关键发现此新媒介中固有的机会。

通过内部原型和浏览来自我们 HoloLens 合作伙伴机构，我们创建了六个问题以帮助您定义共享的方案。 这些问题形成一种框架，不打算详尽说明，以帮助你的方案的重要属性中提取。

### <a name="1-how-are-they-sharing"></a>1.它们如何共享？

多个用户可以进行协作，或教师指导提供给虚拟学生使用虚拟材料时，可能由单个虚拟用户，带领演示文稿，体验会增加复杂性根据的机构的用户具有的级别或可以在一个方案中。

![订婚 holograph 表上的](images/man-and-women-with-holograph-on-table-500px.png)

有很多种，若要共享，但我们发现，其中大多数都分为三类：
* **演示文稿**:当正向多个用户显示相同的内容。 例如：教授向多个学生使用相同的 holographic 材料提供给每个人都提供讲座。 但是教授可以他/她自己的提示和可能不会显示给其他人的说明。
* **协作**:当人员在一起以实现一些常见的目标。 例如：教授出项目提供若要了解如何执行核心修改。 学生配对，并创建共享的技能实验室体验，可将医疗学生核心模型上进行协作和学习。
* **指南**:当一个人帮助人来解决更一一样式交互中的问题。 例如：教授，他/她执行核心外科技能实验室中的共享体验时向学生提供指南。

### <a name="2-what-is-the-group-size"></a>2.什么是组大小？

**一对一**共享体验可以提供强基线，理想情况下可以在此级别中创建在概念证明。 但请注意，与较大 （超过 6 的人） 的组共享会导致难以从技术 （数据和网络） 到社会 (不使用聊天室的影响[多个虚拟形象](https://vimeo.com/160704056))。 在您从复杂性以指数方式增加**小型**到**大型组**。

我们已发现的组的需求可以划分为三个大小类别：
* 1:1
* 小 < 7
* 大型 > = 7

组大小对重要的问题，因为它会影响：
* Holographic 空间中的人员的表示形式
* 对象的小数位数
* 环境的规模

### <a name="3-where-is-everyone"></a>3.每个人都位于何处？

混合现实的强度发挥作用的共享的体验可以发生在同一位置时。 我们将这种**共置**。 相反，分发组和至少一个参与者不在同一物理空间 （如通常是与 VR） 时我们调用的远程体验。 通常它是你的组具有这种情况**同时**归置的和远程参与者 （例如，在会议室中的两个组）。

![与上表 holograph 三人](images/three-people-with-holograph-on-table-500px.png)

以下类别帮助传达用户的位置：
* 共存：所有用户都将在同一物理空间中。
* 远程：所有用户都将在单独的物理空间中。
* 两者：你的用户将是共置和远程空格的组合。

为什么这个问题至关重要，因为它会影响的一些原因：
* 人们的沟通方式？
* 例如：是否应具有虚拟形象？
* 哪些对象他们将看到。 所有对象都共享？
* 是否需要调整以适应其环境？

### <a name="4-when-are-they-sharing"></a>4.何时它们共享？

我们通常认为**同步**体验时共享的体验会想到：我们所有这么做在一起。 但包括由其他人已添加的单个虚拟元素，并且您具有**异步**方案。 请注意或语音备忘录，虚拟环境中的剩余假设。 如何处理你的设计上剩余的 100 虚拟备忘录？ 如果它们是隐私的从几十个用户使用不同级别？

请考虑时这些类别之一作为你的体验：
* **以同步方式**:共享在同一时间的全息版体验。 例如：在同一时间执行技能实验室的两个学生。
* **以异步方式**:共享全息在不同的时间。 例如：执行技能实验室，但在不同时间处理各单独章节的两个学生。
* **同时**:你的用户有时要共享同步但其他情况下以异步方式。 例如：教授评分执行在更高版本的时间和放任说明学生学生的下一步一天的分配。

为什么这个问题很重要的因为它会影响的一些原因：
* 对象和环境暂留。 例如：存储状态，以便它们可进行检索。
* 用户的角度。 例如：可能记住哪些用户了解离开说明时。

### <a name="5-how-similar-are-their-physical-environments"></a>5.相似程度是其物理环境？

两个相同的实际环境，外部归置体验的可能性是精简的用户，除非这些环境是可完全相同。 您更有可能能够**类似**环境。 例如，会议室是类似 — 通常会有位于中心位置的表通过椅子括起来。 生活聊天室，但是，通常是**不同**并且可以包括任意数量的家具的部分中的布局的无限数组。

![上表 holograph](images/holograph-on-table-500px.png)

请考虑共享经验拟合到这两个类别之一：
* **类似**:环境往往具有类似家具、 环境光线和声音，物理空间大小。 例如：教授在讲座 hall A 中，但学生是讲座 hall B.讲座 hall 一个可能有较少椅子比 B，但它们都可能有物理桌上放置全息。
* **不同**:有很大区别家具设置的空间大小、 浅色和声音注意事项中的环境。 例如：教授处于焦点房间，而在填充的学生和教师大型讲座 hall 是学生。

请务必考虑该环境，因为这会影响：
* 用户将如何体验这些对象。 例如：如果您的体验最适合于表和用户有没有表？ 或在平面基底面，但用户混乱的空间。
* 对象的小数位数。 例如：在表上放置 6 英尺人工模型可能颇具挑战性，但核心模型会很好地工作。

### <a name="6-what-devices-are-they-using"></a>6.他们使用什么设备？

现在您通常可能会看到两个之间的共享的体验**沉浸式设备**(这些设备可能在按钮和相对功能方面略有不同，但不是极大地) 或两个**holographic 设备**给定目标在这些设备的解决方案。 但考虑**2D 设备**（移动/桌面参与者或观察者） 将必要的考虑因素，尤其是在的情况下**混合 2D 和 3D 设备**。 了解您的参与者将使用的设备类型非常重要，但不是只是因为它们都具有不同保真度和数据约束和机会，因为用户具有对每个平台唯一的期望。

## <a name="get-started-sharing-experiences"></a>开始共享体验

根据你的应用程序和方案，将各种需求，以实现您所需的体验。 其中包括
* 匹配项进行：若要创建会话，会话中，播发和发现并邀请特定人员，了本地和远程加入您的会话的功能。
* 共享的定位点：能够跨多个设备中常见的本地空间，对齐坐标，因此全息会显示所有人位于同一位置。
* 网络：能够定位，交互，并在同步的人员和全息实时所有参与者。
* 状态存储：若要将全息图特征和位置存储在中间会话联接的空间，请回想一下在更高版本的时间和可靠性对网络问题的功能。


共享体验的关键是他们自己的设备，通过共享坐标对齐在设备上的定位点经常完成上看到相同全息世界上的多个用户。
若要共享的定位点，请使用<a href="https://docs.microsoft.com/azure/spatial-anchors" target="_blank">Azure 空间的定位点</a>:
* 第一次用户将放入全息图。
* 应用程序创建[空间的定位点](coordinate-systems.md)固定世界中准确地说该全息图。
* 定位点可以共享到 HoloLens、 iOS 和 Android 设备通过<a href="https://docs.microsoft.com/azure/spatial-anchors" target="_blank">Azure 空间的定位点</a>。 

使用共享的空间定位点，每个设备上的应用现在有了常见的坐标系统，它们可以在其中放置内容。 现在，应用程序可以确保放置和定位在同一位置全息图。
