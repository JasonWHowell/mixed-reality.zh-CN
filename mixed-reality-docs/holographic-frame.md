# <a name="holographic-frame"></a>Holographic 帧

用户会看到通过其耳机为后盾的矩形视区的混合现实世界。 在 HoloLens，此矩形区域称为 holographic 帧，允许用户查看叠加到其周围现实世界上的数字内容。 设计针对 holographic 框架进行了优化的体验创造了机会，可缓解挑战和增强了混合的现实应用程序的用户体验。

## <a name="designing-for-content"></a>设计与针对内容

通常设计人员都认为需要限制集合范围的用户可以立即看到的内容到他们的体验，影响实际的规模，以确保用户看到完整的对象。 同样带复杂的应用程序的设计器通常重载具有内容、 全息版的帧庞大的用户很难交互和混乱的接口。 设计器创建混合的现实内容需要限制到其体验直接呈现给用户和在其即时视图。 如果物理世界各地用户映射，然后这些图面应被视为用于数字内容和交互的潜在画布。 正确设计的交互和体验中的内容应鼓励用户移动其空间，将定向其注意力转至内容，并帮助请参阅混合现实的全部潜力。

可能是鼓舞人心的移动和应用内的浏览到的最重要技术是对**让用户体验调整**。 用户提供一个短时间内释放任务-与设备。 这可以是简单的空间中放置一个对象和表示允许用户在其周围移动或旁白体验的简介。 这次应该没有任何关键任务或特定于手势 （如空气-点击），而是让用户通过设备需要交互性或应用程序的各个阶段的进展情况之前查看内容适应的目的。 如果这是与设备的用户的第一次这是相当重要，因为它们能够轻松查看内容通过 holographic 帧和全息性质。

### <a name="large-objects"></a>大型对象

通常需要一种体验的内容，尤其是实际内容将会大于 holographic 帧。 通常不适合 holographic 范围内的对象应收缩以适应他们首次引入 （规模较小或最远） 时。 该键**让用户可以查看该对象的完整大小**规模通篇帧之前。 例如，应显示 holographic 大象以完全适合该框架，允许用户以形成一个之前大小调整到空间的动物的整体形状，了解[实际规模](scale.md)附近用户。

具有记住对象的完整大小，用户则具有位置来回移动并查找该对象的特定部分的假定条件。 同样，在沉浸式内容体验，它可以帮助以某种方式来引用该内容的完整大小。 例如，如果体验涉及围绕虚拟房屋的模型的每个步骤，它可能有助于具有较小的玩具房屋大小版本的体验，用户可触发了解他们在房屋的位置。

对于大型对象的设计的示例，请参阅[Volvo 汽车](holographic-frame.md#volvo-cars)。

### <a name="many-objects"></a>许多对象

使用多个对象或组件的体验应考虑使用用户周围的完整空间以避免干扰 holographic 框架直接呈现给用户。 一般情况下，它是好的做法，向内容介绍一种体验缓慢，这是打算向用户提供多个对象的体验与尤其如此。 有了大型对象，该密钥是像**让用户了解的内容布局**中获得的体验，帮助他们空间了解什么是围绕这些内容添加到体验。

一种方法来实现此目的是提供对现实世界的定位点内容的体验中的持久点 （也称为特征点）。 例如，地标可能是在现实世界，如数字内容的显示位置的表或一个数字的对象，如一组数字内容频繁出现的屏幕中的物理对象。 此外可以将对象放置在外围 holographic 帧以鼓励用户以寻找密钥内容，而发现的内容超出外围可以得益[关注主管](holographic-frame.md#attention-directors)。

对象置于外围可以鼓励用户查找到的一侧和这可以得益于注意控制器，如下所述。

## <a name="user-comfort"></a>用户舒适

使用大型对象或多个对象的混合的现实体验，考虑多少头至关重要，颈部移动与内容交互所需。 体验可以分为三个类别在磁头运动方面：**水平**（端到端），**垂直**（向上和向下），或**沉浸式**（水平和垂直）。 如果可能，将限制理想情况下存在与大多数体验发生 holographic 帧的中心中用户的 head 处于中立状态时的大部分交互到水平或垂直类别。 避免导致用户不断地将其视图移到非自然头位置 （例如，始终查找访问密钥菜单交互） 的交互。

![内容的最佳区域是 0 到 35 度下面即将推出](images/optimal-field-of-view-2-750px.png)<br>
*内容的最佳区域是 0 到 35 度下面即将推出*

水平磁头运动是详细[舒适](comfort.md)频繁交互，而垂直移动应该保留的不常见的事件。 例如，涉及较长的水平时间线的体验应该限制垂直磁头运动的交互 （如向下查看菜单）。

请考虑通过将对象放在用户的空间周围鼓励完整正文移动，而不是只是磁头运动。 与移动对象或大型对象的体验也需要特别注意磁头运动，尤其是需要频繁移动沿水平和垂直轴。

## <a name="interaction-considerations"></a>交互注意事项

使用内容，如混合的现实体验中的对象交互不需要限制为用户可以立即看到的内容。 交互可以发生任意位置在周围用户的实际空间，这些交互可帮助鼓励用户四处移动，并浏览体验。

### <a name="attention-directors"></a>注意控制器

指示感兴趣或密钥交互的点上可能会进行用户体验至关重要。 用户注意和 holographic 帧移动可以出细微或笨拙的方式重定向。 请记住以平衡混合现实 （尤其是在一种体验开头） 中包含的免费浏览期间注意控制器以避免庞大的用户。 一般情况下，有两种类型包括： 注意控制器：
* **Visual 控制器：** 若要让用户知道他们应按特定方向移动的最简单方法是提供的直观指示。 这可以通过视觉效果 （例如，用户可以直观地按照向体验的下一部分中的路径） 或甚至作为简单方向箭头。 请注意任何可视的指示器应在用户的环境中，不到全息版的框架或光标附加接地。
* **音频控制器：**[空间声音](spatial-sound-design.md)可以提供以场景 （警报的对象输入体验的用户） 中建立对象或指向空间 （以移动用户的视角看待键对象） 中的特定点的强大方法。 使用音频控制器来指导用户的注意力可能会更微妙、 比 visual 董事会少受侵入。 在某些情况下，它可以是最佳开始音频的主管，然后将上移至 visual 总监如果用户不能识别该提示。 此外可以与 visual 控制器以提供添加强调配对音频控制器。

### <a name="commanding-navigation-and-menus"></a>命令、 导航和菜单

混合的现实体验中的接口在理想情况下紧密与配对他们控制的数字内容。 在这种情况下，自由浮动 2D 菜单通常不是交互的理想选择，很难对用户轻松使用 holographic 框架内。 对于需要界面元素，如菜单或文本字段的体验，请考虑使用[尾随方法](billboarding-and-tag-along.md)一小段延迟后遵循 holographic 帧。 避免锁定到类似于危险警告显示，帧的内容，因为这可以为用户和中断场景中的其他数字对象的深入探讨在意义上有些迷惑。

或者，考虑将放界面元素直接对特定于内容它们控制，允许自然地发生在用户的物理空间周围的交互。 例如，分解复杂菜单为独立的部分。 与每个按钮或组的控件附加到特定对象的交互会影响。 若要进一步这一概念，请考虑使用[种交互对象](interactable-object.md)。

### <a name="gaze-and-gaze-targeting"></a>视线移动和目标的视线移动

Holographic 帧为触发器的交互开发人员提供一种工具，以及评估用户的注意力 dwells 其中。 [注视](gaze.md)是之一[密钥上 HoloLens 交互](interaction-fundamentals.md)，其中的视线移动可以搭配[手势](gestures.md)(如与以无线方式点击) 或[语音](voice-input.md)（允许更短更自然的基于语音的交互)。 在这种情况下，这使得 holographic 帧这两个空间观察数字内容，以及与其进行交互。 如果体验调用与用户的空间 （例如，选择多对象在周围的视线移动 + 手势的用户的空间） 周围的多个对象进行交互时，请考虑将这些对象添加到用户的视图或限制的必要头量移动，以提升[用户舒适](comfort.md)。

提供注视还可用来跟踪用户体验的注意力并查看哪些对象或场景用户的部分付费大多数注意到。 这可以是特别是用于调试的体验，允许分析工具，如热度地图，若要查看用户上最多花费的时间或缺少某些对象或交互。 跟踪的视线移动还可为参与实施体验中提供功能强大的工具 (请参阅[Lowe 的厨房](holographic-frame.md#lowes-kitchen)示例)。

## <a name="performance"></a>性能

正确使用 holographic 帧的是基础[性能质量](understanding-performance-for-mixed-reality.md)体验。 常见的技术 （和可用性） 难题重载具有数字内容，从而导致呈现性能下降的用户的框架。 请考虑改为使用用户周围的完整空间排列数字内容，使用的技术上面所述，以减少呈现的负担，并确保最佳显示质量。

HoloLens holographic 范围内的数字内容也可以搭配[稳定平面](case-study-using-the-stabilization-plane-to-reduce-holographic-turbulence.md)为了获得最佳性能并[全息图稳定性](hologram-stability.md)。
