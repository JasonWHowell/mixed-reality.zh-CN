## <a name="microsofts-ambition-for-mixed-reality"></a>想要实现的混合现实的 Microsoft 的分布

我们生活在两者之间的我们的生活： 物理机和数字。 这两个具有基本的优势，我们利用来补充和扩展我们的能力。 我们可以讨论参加与一位朋友，使用自己的物理感官理解一些喜欢身体语言或我们可以提供了我们讨论的视频-与聊天英里的好友消失的能力。 现在这两个领域和其各自的优势之前, 已从根本上拆分。

**物理世界是原子和物理特性之一。** 我们使用自己的感官做出决策，利用年的已了解行为与对象和我们的环境中的人员进行交互。 尽管这些交互的方便，我们都受我们物理功能和法律的性质。

**数字世界是一种位和逻辑。** 可以轻松地分发信息时，将立即进行计算。 尽管的速度和信息流，交互是太通常限制为小屏幕、 抽象的输入和干扰源。

![HoloStudio](images/holostudio-450px.jpg)<br>
*HoloStudio*

如果我们可以结合使用的物理和数字世界优点？ 这是跨各种混合现实体验的基础： 中等的物理机和数字共存和无缝交互。 组合这些领域生成新的技术与更自然交互的基础 — 在个人计算技术演变而来。

混合现实的范围已分解为开发人员已经开始探索的浸入式和状态显示的机会。 将用户带到沉浸式 （虚拟现实） 体验与数字世界和 holographic （增强的现实） 体验与现实环境中创建数字对象。 但映射到数字世界的现实世界的优点是什么？ 当我们向计算机眼睛，若要查看时，会发生什么情况？

全息背后的基本照相机视觉技术的作用类似于的计算机，以查看你的环境的眼睛对：发生时，对象在世界中，人们周围的更改。 现实生活中上下文的一个数字的了解。 这会产生了巨大的信息，其中的含义，我们将仅开始了解。

### <a name="culminating-core-technologies"></a>通过核心技术

计算通常是一个有形的。 获取我们要告诉我们是以及我们想要的设备。 需要听到扭曲我们的想法和对齐我们说以匹配我们认为计算机。

混合现实，尤其是在现实世界与全息版体验的承诺，为了降低出现与技术交互的负担。 减少认知负载，从而在用户浏览的固有的立即计算的抽象层。 我们可以如何设计体验，不仅利用上下文的理解，而且更加轻松地得出深刻见解，并采取措施？ 两个相关技术还尝试解决此问题：
* **语音**、 语音和会话，方面让用户与计算机通过更自然的方式进行通信，回答机器人通过文本或对话式硬件向发出命令。
* **AI**为提供强大支持提取并将其从日益复杂的数据集的体验。 AI 是一个巨大的主题，而最近进行的设备的依赖计算机视觉、 更自然的数字助理以及向用户推荐的操作提供打下基础。

混合的现实提供了将这些技术组合到单个用户体验的介质。 语音将变为戴上了全息耳机为输入的功能强大、 自然方法。 AI 充当关键的密码器 contextualize 了大量的连接物理和数字世界的信息。 这就是原因 Sataya Nadella 指 HoloLens 作为旗舰版的电脑，它是最终的设备的三个核心技术。 一个平台，用于使人们能够更轻松地与日益增长的复杂性的设备和服务进行交互。

### <a name="less-interface-in-your-face"></a>在你的表盘较少接口

连接到数字世界的现实世界的最终设备可为设计体验适合更自然，而无需繁琐的抽象。 已创建的体验，请考虑：如果删除的抽象的障碍，多少接口为从左？ 您知道用户和其上下文时，将更改应用流中的哪些部分？ 多少菜单和按钮可以保持状态？

例如，考虑一下共享体验混合现实等的 NASA 喷气推进实验室构建科学家 OnSight 工具中。 而不是生成一个系统，可查看火星 （抽象化拖到屏幕或显示的数据） 的数据，这些值存储在数据科学家有效地将它们放在 Mars 的图面上，如会上。

同样，语音和 AI 可以扩展此类的体验中的自然交互。 当数字助理可以看到到周围的世界时，会话将会感到减少很多麻烦。 NASA 的工具中智能机器人应用程序可能从通过火星地质学交换意见时科学家手势和询问有关 this 或 that 用作一个准备就绪 （和对话式） 的信息来源的科学家提取上下文。 知道您的上下文的计算机已准备好直接使用正确的通过一种交互操作最适合您的上下文信息。
